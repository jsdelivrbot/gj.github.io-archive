an array within an array

Our current code stands at 21 lines and 1201 characters, but with one simple (yet very powerful) feature of Ruby, we can get those figures to 22 and 755, respectively.

We briefly touched upon arrays earlier, describing them as [collections, of, comma-separated, values, bounded, by, square brackets]. The values within an array can be any type of data, from strings -- ["Hello", "World!"] -- to integers -- [1, 2, 3, 4, 5] -- to Sonic Pi note values -- [:F4, :C5, :F4].

But what if we could create an array... of arrays?

We're in luck- Sonic Pi and Ruby both allow for multidimensional arrays. We can store the notes from the [F-pattern, E-pattern] array AND the notes from the [F-pattern, E-pattern, D-pattern, A-pattern] array within the same array. How cool is that?

If you said, "So cool," you're absolutely right!

Our new super-array looks something like this:

I've named my array patterns, and I created it with an equals sign. The two second-level arrays (PATTERN 1 and PATTERN 2) are encapsulated within the top-level array and separated by a comma.

Each item stored in an array is assigned an index number. The first item is at index 0, and the last item is at index n-1, where n is the total number of items in the array. If we want to call upon the first item within the patterns array, we can write patterns[0]. That will return the value stored at position 0 within the array, which, in patterns, is the second-level array PATTERN 1. For a further example, in an array named INCEPTION containing 500 items, a call to INCEPTION[499] would return the last item in the array.

Note: calls to positive index numbers, such as INCEPTION[499], count from the beginning of the array. Calls to negative index numbers count from the end. A call to INCEPTION[-1] would return the last entry in the 500-item array, and INCEPTION[-500] would return the first.

Once we create our super-array patterns, we can replace instances of PATTERN 1 and PATTERN 2 in our Sonic Pi code with calls to patterns[0] and patterns[1], respectively.

Reference code below the spinning top.

                # PATTERN 1
patterns = [ [ :F4, :C5, :F4, :F4, :C5, :F4, :F4, :C5, :F4, :C5, :F4, :F4, :C5, :F4, :C5, :F4,   # F-pattern

                # PATTERN 2
             [ :F4, :C5, :F4, :F4, :C5, :F4, :F4, :C5, :F4, :C5, :F4, :F4, :C5, :F4, :C5, :F4,   # F-pattern
               :E4, :C5, :E4, :E4, :C5, :E4, :E4, :C5, :E4, :C5, :E4, :E4, :C5, :E4, :E4, :C5,   # E-pattern
