verbosity

If you're still not quite sure how all those nested loops function in tandem or what's going on when we pass parameters around, let's talk about puts, which helps us figure out what our code is actually doing.

We can manually track what Sonic Pi is going to do when it runs our code, but that involves a ton of effort. To illustrate, here's a step-by-step breakdown of what happens when order[0] passes [0, 1] into patterns_sequence:

  1. patterns_sequence.each becomes [0, 1].each
  2. 0, the first value from [0, 1], is passed into the do / end block and parameterized as index
  3. The central play_pattern_timed method calls patterns[index], which is now patterns[0], which is the F-pattern of notes
  4. play_pattern_timed patterns[index], [0.5] is now play_pattern_timed [F-pattern], [0.5]
  5. After play_pattern_timed completes its first run, patterns_sequence.each passes in its next value, 1
  6. 1 is passed into the do / end block and parameterized as index
  7. The central play_pattern_timed method calls patterns[index] again, which is now patterns[1], the E-pattern of notes
  8. play_pattern_timed patterns[index], [0.5] is now play_pattern_timed [E-pattern], [0.5]
  9. After play_pattern_timed completes its second run, patterns_sequence.each does not find a third value to iterate, and we exit that level of the nested loops, heading back to the repeat.times do level
Whew! Who has time to do all that for an entire song? Not this guy. Luckily, puts allows us to have Sonic Pi do the work for us.

puts is a built-in Ruby method that we can use to print messages to the Sonic Pi log as our code runs. For our present purposes, we can have puts print out the new values of our parameters every time they update, helping us track what's going on while Sonic Pi iterates through order and patterns. We're not going to go into too much detail here, but, briefly, Ruby's string interpolation feature allows us to grab pieces of active code, such as the parameters that are passed between loops, and print them out with puts.

Before we start putsing about, we have to silence all the normal messages Sonic Pi already logs. Go to the Studio tab in Sonic Pi's Preferences and uncheck these three boxes under Logging:

Then, copy use_bpm 175 and the two arrays from Buffer 0 into a new buffer and, below those lines, paste the following code: 

puts "Follow along as the method is continuously\n    updated according to its current parameters:\n"
order.each do |patterns_sequence, repeat, sleep_after|
  puts "order.each do |#{patterns_sequence}, #{repeat}, #{sleep_after}|\n        #{repeat}.times do"
  repeat_counter = 1
  repeat.times do
    unless repeat_counter == 1
      puts "    (Repetition #{repeat_counter})"
    end
    patterns_sequence.each do |index|
      puts "        #{patterns_sequence}.each do |#{index}|\n                play_pattern_timed patterns[#{index}], [0.5]\n            end"
      if index == patterns_sequence[-1]
        puts "    end"
        unless repeat == 1
          if sleep_after == 0
            if repeat_counter == repeat
              puts "end"
            end
          end
        end
      end
      play_pattern_timed patterns[index], [0.5]
    end
    repeat_counter += 1
  end
  unless sleep_after == 0
    puts "    sleep #{sleep_after}\n    end"
  end
  sleep sleep_after
  unless repeat == 10
    puts "Now it's time to iterate order.each\n    and update our three parameters:\n"
  end
end
This is a (very) verbose version of the same eight-line engine we constructed on the previous page. When you press Run in the buffer containing the verbose version, Sonic Pi will start spitting out each line of code as it runs, replacing variables with their current values. Test it out!
